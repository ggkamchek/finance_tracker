# 04. Тест-кейсы

Документ описывает набор ручных проверок для `finance_tracker` и форму фиксации результатов.

## 1. Шаблон фиксации результата

Для каждого кейса заполняются:
- **Ожидаемый результат** - эталонное поведение;
- **Фактический результат** - что получено при выполнении;
- **Статус** - `<PASS/FAIL>`;
- **Комментарий** - причины отклонений (при наличии).

## 2. Критерии завершения тестирования

- выполнены все кейсы `TC-01 ... TC-09`;
- критические сценарии (загрузка, добавление, сохранение, баланс) имеют статус `PASS`;
- не осталось блокирующих дефектов уровня `High/Critical`;
- автотесты проходят командой `go test ./...`.
